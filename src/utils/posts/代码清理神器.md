# Knip - 一键清理项目无用代码

## 简介

在项目开发过程中，随着时间推移，代码库中难免会积累一些不再使用的文件、函数、变量等"死代码"。这些无用代码不仅会增加代码库的复杂度，还可能导致维护困难。**Knip** 是一个强大的静态代码分析工具，能够自动检测并帮助你清理 JavaScript 和 TypeScript 项目中的无用代码。

## Knip 的核心作用

### 1. **检测未使用的文件**
识别项目中没有被任何地方导入的文件。

### 2. **发现未使用的导出**
找出定义在文件中但从未被使用的函数、类、变量等。

### 3. **检测未使用的依赖**
识别 `package.json` 中已安装但未在代码中使用的依赖包。

### 4. **分析项目结构**
深入分析代码之间的依赖关系，精确定位死代码。

### 5. **支持多种框架**
支持 Vue、React、Next.js、Nuxt、TypeScript、ESM、CommonJS 等多种项目类型。

## 安装和配置

### 全局安装

```bash
npm install -g knip
# 或
yarn global add knip
# 或
pnpm add -g knip
```

### 项目级安装（推荐）

```bash
npm install --save-dev knip
# 或
yarn add --dev knip
# 或
pnpm add -D knip
```

### 基础配置

在项目根目录创建 `knip.json` 配置文件：

```json
{
  "entry": ["src/main.js"],
  "project": ["src/**/*.{js,vue,ts,tsx}"],
  "ignore": ["**/*.spec.ts", "**/*.test.ts", "src/types/**"]
}
```

## 常用命令

### 检查项目

```bash
npx knip
```

这会输出所有找到的未使用的文件、导出和依赖。

### 修复未使用的文件

```bash
npx knip --fix
# 或删除
npx knip --fix --delete
```

### 生成报告

```bash
npx knip --reporter json > report.json
npx knip --reporter csv > report.csv
npx knip --reporter sarif > report.sarif
```

### 严格模式

```bash
npx knip --strict
```

此模式下，任何问题都会导致命令失败（非零退出码），适合 CI/CD 流程。

## 实战示例

假设你有一个 Vue 3 项目的结构如下：

```
src/
├── main.js
├── App.vue
├── components/
│   ├── Header.vue
│   ├── Footer.vue
│   └── OldComponent.vue  (已弃用但未删除)
├── utils/
│   ├── helpers.js
│   └── deprecated.js  (未被使用)
└── router/
    └── index.js
```

运行以下命令：

```bash
npx knip
```

可能会输出：

```
Unused files (2):
  src/components/OldComponent.vue
  src/utils/deprecated.js

Unused exports (3):
  src/utils/helpers.js: unusedHelper()
  src/router/index.js: oldRoute
```

然后你可以手动审查这些文件，确认后删除它们：

```bash
npx knip --fix
```

## Vue 3 项目特殊配置

对于 Vue 3 项目，推荐的 `knip.json` 配置：

```json
{
  "entry": ["src/main.js"],
  "project": ["src/**/*.{js,vue,ts,tsx}"],
  "ignore": ["src/types/**", "**/*.spec.ts"],
  "tsConfigPath": "tsconfig.json"
}
```

## 与 package.json 集成

在 `package.json` 中添加脚本：

```json
{
  "scripts": {
    "lint": "eslint src",
    "check-unused": "knip",
    "check-unused:fix": "knip --fix",
    "check-unused:strict": "knip --strict"
  }
}
```

## 最佳实践

### 1. **定期运行**
在 CI/CD 流程中集成 Knip，定期检查未使用的代码：

```yaml
# GitHub Actions 示例
name: Code Quality
on: [push, pull_request]
jobs:
  knip:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
      - run: npm install
      - run: npx knip --strict
```

### 2. **分阶段清理**
不要一次性清理所有代码，分阶段进行：
- 先生成报告，人工审查
- 确认无误后再删除
- 提交到 Git 后进行测试

### 3. **添加忽略规则**
某些文件可能看起来没有被使用，但实际上是通过动态导入或其他方式引用的，需要在配置中忽略：

```json
{
  "ignore": ["src/plugins/**", "src/directives/**"]
}
```

### 4. **关注导出而非文件**
有时一个文件中有多个导出，只有部分被使用，Knip 能精确识别未使用的导出，帮助你只删除需要的部分。

| 检测未使用导出 | ✅ | ⚠️ (插件) | ❌ |
| 检测未使用依赖 | ✅ | ❌ | ✅ |
| 自动修复 | ✅ | ✅ | ❌ |
| 性能 | 快速 | 快速 | 一般 |

## 注意事项

⚠️ **使用前备份**：在使用 `--fix --delete` 前，确保你的代码已提交到版本控制系统。

⚠️ **人工审查**：不要盲目删除所有报告的代码，某些动态导入、特殊框架特性可能被误认为是无用代码。

⚠️ **配置精确性**：确保 `entry` 和 `project` 配置准确，否则可能误报大量的"未使用代码"。

## 总结

Knip 是一个高效的代码清理工具，能够帮助你：
- 🧹 自动识别和清理无用代码
- 📊 生成详细的代码使用报告
- 🔄 集成到 CI/CD 流程中自动化检查
- 🎯 改进代码质量和可维护性

通过定期使用 Knip，你可以保持代码库的整洁，提高项目的可维护性和团队的开发效率。
